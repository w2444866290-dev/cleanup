# cleanup-disk.sh — UI 现状汇总

> 用途：作为 UI 美化的参考底稿，记录脚本当前所有用户可见场景的样式与行为。
>
> 对应脚本：`~/cleanup-disk.sh`

---

## 0. 通用样式构件

### 颜色常量（仅在 `isatty(stdout)` 时启用，否则全为空字符串）

| 变量 | 用途 | 实际值 |
|---|---|---|
| `C_BOLD` | 强调 | `\033[1m` |
| `C_DIM` | 暗显 | `\033[2m` |
| `C_RED` | 错误 | `\033[31m` |
| `C_GREEN` | 成功 / 选中 / 焦点 | `\033[32m` |
| `C_YELLOW` | 大小数值 / 警告 | `\033[33m` |
| `C_BLUE` | 标题 | `\033[34m` |
| `C_CYAN` | 分类标记 / 进度 | `\033[36m` |

### 文字符号

| 元素 | 符号 | 颜色 |
|---|---|---|
| 区块标题 | 无前导符号 | `C_BOLD + C_BLUE` |
| 警告 | `!` | `C_YELLOW` |
| 成功 | `✓` | `C_GREEN` |
| 失败 | `✗` | `C_RED` |
| 提示注释 | 行首两空格 + 内容 | `C_DIM` |
| 列表光标 | `▶` | `C_BOLD + C_GREEN` |
| 复选框：未选 / 已选 | `[ ]` / `[✓]` | 已选用 `C_GREEN` |
| 分类点（确认页） | `●` | `C_BOLD + C_CYAN` |
| 按钮 | `[ 文本 ]` | 焦点态 `C_BOLD + C_GREEN`；空选态 `C_DIM` |
| 行动指示 | `➜` | 跟随按钮颜色 |

### 分隔线

| 函数 | 字符 | 长度 |
|---|---|---|
| `hr()` | `=` | 64 |
| `hr2()` | `-` | 64 |
| `header(t)` | `hr` + 标题 + `hr` | — |
| `draw_footer_line(label)` | `─` 两侧填充 + `label` | 居中，总宽约 64 |

### 大小 / 磁盘 readout

```
  总容量 460Gi   已用 326Gi   可用 110Gi   占用 75%
```

- `总容量 / 已用 / 占用` 标签 `C_DIM` + 数值默认色
- `可用` 标签 `C_DIM` + 数值 `C_BOLD + C_GREEN`（最显眼）
- 来自 `print_disk_status`

**`human_kb` 输出格式**

- `< 1024 KB` → `<n>K`
- `< 1024 MB` → `<n.n>M`（一位小数）
- `≥ 1 GB` → `<n.n>G`（一位小数）

## 1. 扫描阶段（脚本启动后到菜单前）

### 1.1 启动横幅

```
================================================================
磁盘清理 — 扫描中
================================================================
  总容量 460Gi   已用 326Gi   可用 110Gi   占用 75%

  正在扫描可清理项，~/go/pkg 较大可能需 ~40 秒...

```

- `header()` 渲染标题
- 一行 `print_disk_status`
- `note()` 显示扫描预告

### 1.2 进度条 — 单步扫描（快项）

单行原地刷新，无 spinner：

```
  [12/29] ████████████░░░░░░░░░░░░  41%  Yarn 缓存
```

- 来自 `scan_step()`，自增 `SCAN_STEP` 后覆盖打印同一行
- 实心比例 = `SCAN_STEP / SCAN_TOTAL`
- 此处只显示描述，不显示路径

### 1.3 进度条 + spinner（慢项：du 大目录、长 find）

单行刷新（10 Hz）：

```
  [18/29] ████████████████████░░░░  62%  ⠹ /Users/bytedance/go/pkg  15s
```

- 后台进程通过 `/dev/tty` 刷新
- spinner 循环：`⠋ ⠙ ⠹ ⠸ ⠼ ⠴ ⠦ ⠧ ⠇ ⠏`
- 已用秒数 `≥ 2s` 时才出现，黄色
- 路径用 `C_DIM` 暗显
- 切目录时旧 spinner 自动停掉、新 spinner 启动

### 1.4 扫描完成清空

进入菜单前调用 `scan_done()` —— `\r\033[K` 把最后一行刷干净，不留进度条残痕。

## 2. 主菜单（`STATE=select`）

### 2.1 正常态（有可清理项）

进入 alt screen + 隐藏光标 + 关闭回显。整屏布局：

```
================================================================
磁盘清理 — 选择要清理的项
[开发工具缓存(3)] [应用缓存(3)] [应用数据 (慎清)(2)] [重型可选 ⚠(1)] [GVM 旧 Go 版本(6)] [不常用 App (180天+ 未启动)(6)] [超大 App (≥500MB)(15)]
================================================================


▶ [ ]  [   2.6G]  gopls / goimports 缓存
  [ ]  [   2.4G]  npm 缓存 (~/.npm)
  [ ]  [   2.8G]  Yarn 缓存


────────────────────────────── 共 3 项 ──────────────────────────────

  [ 开始清理 ➜ 未选项 ]

← → 切 tab   ↑↓ 移动/翻页 (↓ 到底进入按钮)   Space/Enter 勾选   a 全选当前tab   s safe   c 清空   q 退出
```

**结构（从上到下）**：

1. 顶部 `=` 线（64 字符，`C_BOLD + C_BLUE`）
2. 大标题 `磁盘清理 — 选择要清理的项`
3. Tabs 栏：`[分类名(项数)]` 空格分隔
   - 当前 tab：`C_BOLD + C_GREEN`
   - 其他 tab：`C_DIM`
4. 底部 `=` 线
5. 空行
6. 上方"其余 N 项 ↑"指示（`C_DIM`）；不在首页时显示，否则空行占位（保持 UI 高度稳定）
7. 列表区：`≤ 7` 行，每行 `marker  [✓/ ]  [size]  desc`
   - marker：`FOCUS=list` 且当前光标行 → `▶ `（绿加粗）；否则两个空格
   - size 数值黄色
8. 下方"其余 N 项 ↓"指示 / 空行占位
9. 空行 → 分页脚 → 空行
   - 分页脚 `── 共 N 项 ──` 居中，两侧 `─` 填充到约 64 宽，`C_DIM`
10. 开始清理按钮（4 种态见下）
11. 空行
12. 快捷键提示行（单行，`C_DIM`，无装饰）

### 2.2 「开始清理」按钮的 4 种态

| 选中数 | FOCUS | 渲染 |
|---|---|---|
| `0` | list | `  [ 开始清理 ➜ 未选项 ]` — 全 `C_DIM` |
| `0` | button | `▶ [ 开始清理 ➜ 未选项 ]` — `▶` 绿加粗，文本仍 `C_DIM` |
| `> 0` | list | `  [ 开始清理 ➜ 已选 5 项, 30.5G ]` — `C_GREEN`，无加粗 |
| `> 0` | button | `▶ [ 开始清理 ➜ 已选 5 项, 30.5G ]` — `▶` + 文本均 `C_BOLD + C_GREEN` |

### 2.3 空状态（`TOTAL=0`，扫不到任何可清理项）

不进 TUI，直接打印一行：

```
  ✓ 没有发现可清理项，磁盘已经很干净了 🎉
```

随后 `exit 0`。

## 3. 二次确认（`STATE=confirm`）

```
================================================================
二次确认 — 以下将被清理 (共 3 项)
================================================================


● 开发工具缓存
  [   2.6G]  gopls / goimports 缓存

● 应用数据 (慎清)
  [   6.7G]  壁纸缓存 (macOS 自动重建)

● GVM 旧 Go 版本
  [  46.2G]  go1.16  — gos+pkgsets, 用 gvm uninstall 卸载

----------------------------------------------------------------
  预计释放: 55.5G
----------------------------------------------------------------

  [ 返回修改 ]     ▶ [ 开始清理 ]

← → 切换按钮   Enter 确认   q/Esc 返回修改
```

**结构**：

1. 顶部 `=` 线（`C_BOLD + C_YELLOW`，警示色）
2. 标题 `二次确认 — 以下将被清理 (共 N 项)`（`C_BOLD + C_YELLOW`）
3. 底部 `=` 线
4. 选中项按 `CATS` 分组展示
   - 分类首行：`● <分类名>`（`C_BOLD + C_CYAN`）
   - 项目行：`  [size]  desc`
5. `-` 分隔线（`C_BOLD + C_GREEN`）
6. `预计释放: X G`（`C_BOLD + C_GREEN`）
7. `-` 分隔线
8. 两个按钮：`[ 返回修改 ]` 和 `[ 开始清理 ]`
   - 当前按钮 = `▶ ` 前缀 + 加粗（返回 = `C_RED`，开始 = `C_GREEN`）
   - 非当前按钮 = `C_DIM`
   - 默认 `CONFIRM_BTN=1`（光标默认在「开始清理」上）
9. 快捷键提示行（`C_DIM`）

## 4. 执行阶段（在确认页按 Enter 后）

退出 alt screen，回到正常终端，逐项打印进度：

```
================================================================
开始清理
================================================================
  [1/3] gopls / goimports 缓存 ... ✓
  [2/3] 壁纸缓存 (macOS 自动重建) ... ✓
  [3/3] go1.16  — gos+pkgsets, 用 gvm uninstall 卸载 ... ✓
```

**结构**：

1. `header "开始清理"`（`=` 线 + 标题 + `=` 线）
2. 每项一行：`  [N/M] <desc> ... `（不换行）
   - 成功后追加 `✓`（`C_GREEN`），换行
   - 失败后追加 `✗`（`C_RED`），`FAIL++`，换行
3. 命令在 subshell 中执行：`</dev/null` 关掉 stdin、`set +u` 防止 source 外部脚本爆栈

---

## 5. 完成阶段（执行结束后）

```
================================================================
完成
================================================================
  总容量 460Gi   已用 271Gi   可用 165Gi   占用 62%
  ✓ 本次释放约 55.2G

  提示：移到废纸篓的文件/App 在 Finder 清空废纸篓后才会真正腾出空间。
```

**结构**：

1. `header "完成"`
2. `print_disk_status`（再打一次磁盘状态，便于前后对比）
3. 释放总量：
   - `FREED_KB > 0` → `ok "本次释放约 X G"`（带 `✓`）
   - `FREED_KB ≤ 0` → `note "本次未观测到空间增加..."`（暗色，提示原因）
4. 如 `FAIL > 0`：`warn "$FAIL 项执行返回非 0（可能文件被占用）"`
5. `note "提示：移到废纸篓的文件/App 在 Finder 清空废纸篓后才会真正腾出空间。"`

## 6. 取消路径

用户在主菜单按 `q / Esc`，或在 confirm 页选「返回修改」后再 `q` —— 退出 alt screen 后单行：

```
  已取消，未执行任何清理。
```

（`note()` 输出，`C_DIM`），然后 `exit 0`。

## 7. 键盘交互速查

### 主菜单 `select` 状态

| 按键 | `FOCUS=list` | `FOCUS=button` |
|---|---|---|
| `←` | 切到上个 tab，FOCUS 重置为 list | 同左 |
| `→` | 切到下个 tab，FOCUS 重置为 list | 同左 |
| `↑` | 列表内上移（含翻页） | 焦点回到列表最末项 |
| `↓` | 列表内下移；末项再 `↓` → `FOCUS=button` | 不动 |
| `Space` | 勾选 / 取消当前项 | 无效 |
| `Enter` | 同 `Space`（勾选当前项） | 进入 confirm（若已选 > 0） |
| `a` / `A` | 全选 / 反选**当前 tab** 内所有项 | 同左 |
| `s` / `S` | 全局 safe 预设（排除重型/旧文件/App/GVM/慎清类） | 同左 |
| `c` / `C` | 全局清空所有勾选 | 同左 |
| `q` / `Q` / `Esc` | 退出 | 同左 |

### `confirm` 状态

| 按键 | 行为 |
|---|---|
| `←` / `→` | 在 `[ 返回修改 ]` ↔ `[ 开始清理 ]` 之间切换 |
| `Enter` | 按所选按钮执行 |
| `q` / `Q` / `Esc` | 返回主菜单（不退出脚本） |

## 已知边界场景

| 场景 | 当前行为 | 备注 |
|---|---|---|
| 信号中断（Ctrl-C） | `trap 'restore_tui; exit 130' INT TERM`，静默退出 | 无定制 UI |
| 进度条 100% 到进入 alt screen 之间 | 视觉一闪而过 | 无可见过渡 |
| `bash -n` 等语法/启动错误 | 直接由 shell 报错 | 未做包装 |
| 终端不支持 `tput smcup` | fallback 到 `printf '\033[?1049h'` | 结果一致 |
| 终端宽度 < 120 字符 | tabs 栏自然 wrap 到第二行 | 视觉略乱但功能正常；未做横向滚动 |
| `set -u` 与外部 source 冲突 | 执行循环已用 subshell + `set +u` 隔离 | 详见 §4 |
